## 2. QA 风险清单

### 2.1 状态不一致风险

| 编号 | 风险 | 场景 | 应对 |
|------|------|------|------|
| Q-1 | 外部 API 成功但数据库写入失败 | 内容生成/视频生成/发布 | Service 层事务性操作，外部调用成功后立即写库 |
| Q-2 | 视频生成中用户刷新页面 | 进度丢失 | 状态存数据库，刷新后根据 providerJobId 继续轮询 |
| Q-3 | Cron 执行时项目状态已变 | 并发 | Cron 查询条件包含状态检查 |

### 2.2 重复提交风险

| 编号 | 风险 | 场景 | 应对 |
|------|------|------|------|
| Q-4 | 连续点击"生成内容" | 创建多个 ContentPlan | 前端 loading 禁用 + API 检查状态 |
| Q-5 | 连续点击"发布" | 重复发布 | 检查 Publication 是否已存在 |
| Q-6 | Cron 重复处理同一项目 | 重复分析 | 状态检查 + 已 ANALYZED 跳过 |

### 2.3 第三方 API 失败

| 编号 | 风险 | 场景 | 应对 |
|------|------|------|------|
| Q-7 | OpenAI 返回非 JSON | 内容生成 | Provider 层解析 + 重试 + 明确错误 |
| Q-8 | 即梦返回成功但无 videoUrl | 视频完成 | Provider 层校验返回数据完整性 |
| Q-9 | TikTok 发布成功但未返回 videoId | 发布 | 发布后查询确认 |
| Q-10 | 即梦视频 URL 已过期 | 发布时 | 发布前检查 URL 有效性 |
| Q-11 | TikTok Token 过期 | 发布/拉数 | 自动 refresh → 失败则提示重新授权 |

### 2.4 数据完整性风险

| 编号 | 风险 | 场景 | 应对 |
|------|------|------|------|
| Q-12 | 用户编辑内容后未保存 | 中途离开 | 前端 beforeunload 提示（MVP 可选） |
| Q-13 | TikTok 发布成功但写库失败 | 网络问题 | 先写库再响应，失败则用户手动重试 |
| Q-14 | 删除项目时有进行中的视频生成 | 数据不一致 | 级联删除，即梦端任务自然过期 |

### 2.5 MVP 范围控制

| 编号 | 风险 | 应对 |
|------|------|------|
| Q-15 | 视频编辑需求蔓延 | 明确不做编辑，只做预览确认 |
| Q-16 | 多账号管理需求 | MVP 只一个 TikTok 账号 |
| Q-17 | 批量操作需求 | MVP 逐条操作 |
| Q-18 | 历史对比需求 | MVP 只单条分析 |
