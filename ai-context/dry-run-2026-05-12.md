粗估：**首轮探索 token 直降 80%**，长 service 二次读取 token 直降 ~85%。

## 结论

| 评估维度 | 结果 |
|----------|------|
| Top-3 命中率（人工判定是否最相关） | 3/3 任务 = 100% |
| 路由识别 | 3/3 任务都自动列出了相关 API endpoint + HTTP 方法 |
| 大文件警告 | 自动标 ⚠（≥800 行的 `video-service.ts` / `ai-video-workflow-demo-data.ts` 都标了） |
| 关键 symbol 暴露 | 暴露了 `dispatchVideoForBrief` / `processPendingFinalVideos` / `claimStitchTask` 等真正会被 jump-to 的函数名 |
| 端到端耗时 | 每条 query 0.25 - 0.30 秒，零 LLM 调用 |
| 误报 / 噪声 | 任务 2 的"shared"和"tests" area 出现在命中 area 列表里属于 false positive，但 Top-10 排名合理；任务 3 的 walkthrough 系列脚本是次要相关，不影响主决策 |

**判定：workflow 可以正式接入开发使用。**

任何 AI agent 接入此 repo 后，按 [`.cursor/rules/ai-context-workflow.mdc`](../.cursor/rules/ai-context-workflow.mdc) 规则操作即可。详细场景查 [`docs/AI_CONTEXT_WORKFLOW.md`](../docs/AI_CONTEXT_WORKFLOW.md)。
