# 投资人 Demo 页（/showcase）国内可访问部署方案

> 背景：`/showcase` 现在部署在 Vercel，**国内访问慢/经常打不开**（Vercel 默认线路在大陆不稳定）。
> CEO 要求：**把这一页单独做成国内能稳定打开的网页**给投资人看。
>
> 本文只针对「单独这一页 demo」的对外可访问问题，给出方案对比与推荐。
> 整套 App 在大陆的完整部署（火山 ECS + RDS + TOS + Ark）见 [docs/CHINA_DEPLOYMENT.md](CHINA_DEPLOYMENT.md)，二者不冲突。

## 1. 现状约束（决定能不能"纯静态"）

`/showcase` 目前有两处服务端依赖，做"单独静态站"前必须先处理：


| 依赖                     | 位置                                                                          | 影响           | 静态化处理                                                                                                                                                                             |
| ---------------------- | --------------------------------------------------------------------------- | ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 登录态 `getServerSession` | [src/app/(public)/showcase/page.tsx](../src/app/(public)/showcase/page.tsx) | 决定 CTA 文案/跳转 | 给投资人看的版本写死 `isAuthenticated=false` 即可，不需要登录                                                                                                                                       |
| 体验申请表单                 | `POST /api/demo/real-footage-ads/waitlist`（Prisma 写库）                       | 表单提交需要后端     | **已处理**：构建时注入 `NEXT_PUBLIC_STATIC_DEMO=true`，表单走 mock 提交（保留 UI、假成功），不连任何后端；线上 Vercel 不设该变量、真实接口不受影响。见 [pet-waitlist-form.tsx](../src/app/(public)/showcase/pet-waitlist-form.tsx) |


**大文件**（影响国内首屏速度，必须配 CDN）：

- 讲解视频 `aivora-pet-content-kit-walkthrough-60s-16x9.mp4` ≈ **35 MB**
- `public/demo/pet/` 图片合计 ≈ **25 MB**（多张 1–2 MB PNG）

建议上线前再压一轮：视频转 H.264 较低码率压到 ~10–15 MB；图片转 WebP，缩略图按显示尺寸出图（封面 600px、头像 128px 足够）。

## 2. 方案对比


| 方案                                                           | 国内速度           | 是否需 ICP 备案 | 上线速度                   | 适用                     |
| ------------------------------------------------------------ | -------------- | ---------- | ---------------------- | ---------------------- |
| **A. 复用火山全栈部署**                                              | 快              | 需要（大陆节点）   | 慢（依赖 RDS/TOS/Ark + 备案） | 长期，且 demo 要带登录/表单/真实生成 |
| **B. 静态导出 + 大陆对象存储 + CDN**（阿里云 OSS / 腾讯 COS / 火山 TOS 静态网站托管） | 最快             | 需要（绑大陆域名时） | 中（卡在备案 7–20 天）         | 长期对外、已有/愿走备案域名         |
| **C. 静态导出 + 香港/海外节点**（香港区 OSS/COS，或一台香港轻量服务器 + Nginx）        | 较快（比 Vercel 稳） | **免备案**    | 最快（1 天内）               | 马上要给投资人看、暂无备案          |
| D. Cloudflare Pages / Netlify / Vercel                       | 不稳定            | 否          | 快                      | ❌ 不建议作为给投资人的主链路        |


## 3. 推荐

- **马上要给投资人看、还没备案 → 选 C。** 把 `/showcase` 做成静态站，传到**香港区对象存储 + CDN**（阿里云/腾讯都有香港 region，免 ICP），或一台香港轻量应用服务器跑 Nginx。1 天内可上线，国内访问明显比 Vercel 稳。
- **要长期对外、愿意走备案 → 选 B。** 静态站放**大陆 OSS/COS + CDN**，绑已备案域名，国内体验最佳、成本最低（纯静态 + CDN 流量费）。
- **demo 之后还要带登录/表单/真实生成 → 选 A**，并入 [CHINA_DEPLOYMENT.md](CHINA_DEPLOYMENT.md) 的火山全栈。

> 备案是唯一的"长周期"变量：要绑大陆域名指向大陆节点，**先去申请 ICP 备案**（约 7–20 天），其它都可当天搞定。先用方案 C 的临时链接给投资人，备案下来再切方案 B。

## 4. 方案 C 落地步骤（已选定，阿里云香港 OSS + CDN，免备案）

> 目标：保证大陆能稳定打开 `/showcase` 链接。当前整个 App 含 API 路由 + NextAuth，**不能整体 `output: 'export'`**，所以用「快照式」静态导出：复用真实生产构建，把 `/showcase` 的 SSR HTML + `_next` 静态资源 + 引用到的图片/视频，打包成自包含静态目录，样式/字体与线上 100% 一致。

### 4.1 生成静态站（开发侧，一条命令）

```bash
bash scripts/build-showcase-static.sh
```

产物在 `showcase-static/`（约 82 MB，已加入 `.gitignore` 不入库），结构：

```
showcase-static/
├── index.html              # /showcase 的 SSR 快照
├── favicon.ico
├── _next/static/...         # JS / CSS / 字体（hash 命名）
├── demo/pet/...             # 图片
└── generated/*.mp4          # 讲解视频 + 4 个证据片段
```

本地预览确认无误：`cd showcase-static && python3 -m http.server 4500`，浏览器开 `http://127.0.0.1:4500/`。

> 表单已是 mock（脚本内置 `NEXT_PUBLIC_STATIC_DEMO=true`），点击「提交体验申请」会显示成功提示但不发任何请求。

### 4.2 阿里云控制台（CEO 侧，已有账户）

1. **建 Bucket**：OSS 控制台 → 创建 Bucket → **地域选「中国香港」(`oss-cn-hongkong`)** → 读写权限 **公共读**。
2. **上传内容**：把 `showcase-static/` 目录里的**所有内容**（注意是「内容」，`index.html` 要在桶根目录，不是再套一层文件夹）上传到 Bucket 根目录，保持 `_next/`、`demo/`、`generated/` 的目录结构。
3. **静态网站托管**：Bucket → 数据管理 → 静态页面 → 默认首页和默认 404 都填 `index.html` → 保存。
4. **接 CDN**：阿里云 CDN（或 DCDN）→ 添加域名 → 源站「OSS 域名」选刚建的桶 → **加速区域选「全球（不含中国大陆）」或「仅海外及港澳台」**。这是免 ICP 备案的关键（一旦勾选含「中国大陆」就必须备案）。
5. **域名**：
  - 最快：直接用 CDN 分配的**默认 CNAME 域名**给投资人，完全不碰备案。
  - 可选：绑自有域名 —— 因加速区域不含大陆，**也不需要备案**，在 DNS 加一条 CNAME 指向 CDN 即可。
6. **HTTPS**：CDN 控制台申请免费证书 → 开启强制 HTTPS。
7. **缓存 / 压缩**：开启 Gzip/Brotli；`_next/`、`demo/`、`generated/` 这类带 hash 或不常变的资源缓存 30 天，`index.html` 缓存设短（如 5 分钟）方便更新。
8. **验证**：用大陆网络打开 CDN 链接，确认页面能开、视频能播、表单能弹「已收到」。

### 4.3 以后更新内容

改完代码/文案 → 重跑 `bash scripts/build-showcase-static.sh` → 覆盖上传 `showcase-static/` 内容 → 在 CDN 控制台**刷新 `index.html`**（及有变动的资源 URL）即可。

### 4.4 备注

- OSS 香港直链也能开（用 Bucket 默认外网域名），但**强烈建议挂 CDN**，大陆访问明显更稳更快。
- 35 MB 讲解视频 + ~25 MB 图片偏大；若大陆首屏仍慢，按第 5 节做一轮压缩再上传。
- 想长期对外、愿走备案 → 之后把同一份 `showcase-static/` 换到**大陆 OSS + CDN + 已备案域名**（方案 B），体验最佳、成本最低。

## 5. 后续可选优化

- 视频额外出一个 9:16 竖版/更低码率版本，移动端国内加载更快。
- 图片统一转 WebP + 响应式 `srcset`。
- 给静态站加一个简单的访问统计（如自建 + 对象存储日志，或国内可用的统计 SDK）。
