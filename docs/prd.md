# ReelForge — 产品需求文档 (PRD)

> 版本: 2.0 | 更新时间: 2026-04-06
> 范围: TikTok AI 短视频自动化平台 MVP

---

## 1. 项目目标

构建一个**中文** TikTok AI 短视频自动化平台 MVP，实现从「输入关键词」到「视频发布 + 数据分析反馈」的完整闭环。

面向个人创作者或小团队使用，目标是大幅降低 TikTok 短视频内容的生产成本和时间。

## 2. 核心业务闭环

```
关键词输入 → AI 内容生成 → AI 视频生成 → 人工审核确认 → TikTok 发布 → 12h 自动拉数 → AI 分析反馈
```

## 3. MVP 功能范围

| 编号 | 功能 | 说明 |
|------|------|------|
| F-1 | 关键词/方向输入 | 用户输入中文关键词或产品方向描述 |
| F-2 | AI 内容生成 | 调用 OpenAI 生成：中文脚本、视频提示词、TikTok caption、hashtags、内容角度建议 |
| F-3 | AI 视频生成 | 调用即梦（Seedance via 火山方舟）生成 AI 视频 |
| F-4 | 视频预览与确认 | 用户在平台内查看视频，确认或拒绝 |
| F-5 | TikTok 发布 | 用户确认后，平台通过 TikTok API 发布视频 |
| F-6 | 自动数据回收 | 发布 12 小时后，Cron 自动拉取 TikTok 播放/互动数据 |
| F-7 | AI 分析报告 | 基于数据生成表现分析、方向建议、优化建议 |
| F-8 | 项目列表与状态 | 用户可查看所有视频项目及其当前状态 |
| F-9 | TikTok 账号绑定 | OAuth 授权连接 TikTok 账号 |

## 4. 明确不做（MVP 排除项）

| 项目 | 理由 |
|------|------|
| 用户注册/登录系统 | MVP 阶段个人使用，用简单密码保护即可 |
| 多用户/团队协作 | 不做 SaaS |
| 多平台发布（Facebook/Instagram/YouTube） | 只做 TikTok |
| 多视频生成商切换 | 只用即梦/Seedance |
| 视频编辑/剪辑 | 不做编辑器，只做预览确认 |
| 批量自动生成 | 用户逐条操作 |
| 高级推荐引擎 | 简单 AI 建议足够 |
| 支付/订阅 | MVP 不涉及 |
| 复杂 BI 报表 | 基础指标即可 |
| A/B 测试 | 后续再做 |
| 复杂权限系统 | 不需要 |
| 消息队列（Redis/Celery） | 不需要 |

## 5. 用户故事

- **US-1**：作为用户，我输入一个中文关键词，系统在数秒内返回完整的内容方案（脚本 + caption + hashtags + 角度建议），我可以查看和编辑
- **US-2**：作为用户，我确认内容后一键触发视频生成，并能看到生成进度
- **US-3**：作为用户，我可以预览生成的视频，满意后一键发布到 TikTok
- **US-4**：作为用户，我可以在列表中看到所有视频项目和各自的状态（生成中/待确认/已发布/已分析）
- **US-5**：作为用户，视频发布后平台自动拉取数据并给出分析报告，帮我判断方向好坏
- **US-6**：作为用户，我在设置页绑定 TikTok 账号，之后发布流程无需重复授权

## 6. 验收标准

- 从输入关键词到内容生成 < 10 秒
- 视频生成在即梦 API 允许范围内完成（预计 1-5 分钟）
- 视频成功发布到 TikTok 且可在 TikTok 上访问
- 12 小时后数据自动回收，包含至少：播放量、点赞数、评论数、分享数
- 分析报告包含：表现评价、方向建议、优化建议
- 任何步骤失败可重试，不丢失已有数据
- 页面文案和交互全部使用中文

## 7. 关键风险

| 风险 | 等级 | 应对策略 |
|------|------|---------|
| TikTok Content Posting API 审批周期长 | **高** | 先用 Mock 跑通全链路，TikTok 发布作为最后接入项。立即开始申请。 |
| 即梦/Seedance API 调用限制和成本 | 中 | 先调研计费模型，MVP 控制调用量 |
| TikTok Analytics API 数据获取范围有限 | 中 | 先确认 API 能返回哪些指标 |
| 即梦生成的视频质量不可控 | 中 | 保留人工审核环节 |
| OpenAI API 中文 token 消耗较大 | 低 | 用 gpt-4o-mini 控制成本 |
| TikTok API 在中国网络下的可访问性 | **待确认** | 部署在 Vercel 上有海外节点，但开发调试可能受影响 |

## 8. 待确认项

- [ ] 即梦 API 的具体接口文档、调用限制、计费模型
- [ ] TikTok Developer 账号的审批流程和预计时间
- [ ] TikTok Content Posting API 对视频格式/大小的要求
- [ ] TikTok API 能获取哪些 Analytics 指标
- [ ] 现有 Python 代码的处理方式（保留/归档/删除）
- [ ] Vercel 计划选择（Hobby vs Pro，影响 Cron 频率和 Function 超时）

## 9. 与前版本的关键变化

| 维度 | 前版本（Python 原型） | 当前版本（Next.js MVP） |
|------|---------------------|----------------------|
| 定位 | B2B 视频服务系统 | 个人/小团队自动化工具 |
| 技术栈 | Python + FastAPI + SQLAlchemy | Next.js + TypeScript + Prisma |
| 数据库 | SQLite | Neon Postgres |
| 部署 | 未定 | Vercel |
| 前端 | 无 | Next.js + Tailwind + shadcn/ui |
| 视频生成 | Mock | 即梦/Seedance (真实接入) |
| TikTok 发布 | 不做 | 核心功能 |
| 数据分析 | 基础 | AI 驱动分析 |
