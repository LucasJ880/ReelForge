# Buddy API 视频 Provider 子循环

状态：**已登记，未启动**。启动条件：主计划 Phase 1 / GATE 1 完成。截止条件：必须在主计划 Phase 5b 真实金丝雀前完成 SL-C。

## 不变量

- Buddy 是新增 provider，不替换或删除 BytePlus Seedance。
- 业务代码只依赖 `VideoProvider`；provider 选择仅来自全局默认与批次级覆盖。
- Buddy 密钥只在环境/托管 secret 中存在，禁止进入代码、日志、报告。
- `confirmed_unit_price` 未由人工配置前，Buddy 真实调用累计预算上限 $20；SL-A 契约探明上限 $5。
- 幂等能力确认前，仅可重试明确未产生计费的连接失败与 4xx 校验错误；超时/5xx 转人工。
- VideoJob 对 Buddy/BytePlus 都必须留存 provider、provider job id、单价快照、请求与完成时间戳。

## 顺序

1. **SL-A 契约探明**：先提交信息索取清单；文档到手后再用 ≤$5 完成成功、校验失败、完整异步生命周期探明；数据位置不明或在中国大陆时停 Gate。
2. **SL-B Adapter/mock**：实现 Buddy adapter、真实响应 fixture、BuddyMock、失败分类、internal 批次级选择与成本记账；零真实调用验收。
3. **SL-C 双轨对比**：人工确认价格与预算后，3 模板 × 10 组 × 2 provider，共 60 条；比较质量、成功率、重试率、P50/P95 与失败摊销成本。
4. **SL-D 并发与切换**：BuddyMock 500 批量；书面确认并发后才做 30–50 条真实并发；GO 后配置切换并演练回退。
