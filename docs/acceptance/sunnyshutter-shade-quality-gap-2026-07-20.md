# SunnyShutter Shade 质量差距分析（2026-07-20）

> 对比对象：我们的 `ss-shade-01-branded.mp4`（v1 批次 #1，Seedance Fast VIP I2V）
> vs 同行金标 `tmp/sunnyshutter-shade-refs/peer-quality-shade-15s.mp4`（真人实拍斑马帘 15s）。
> 结合 CEO WeChat 反馈（logo 重合 / 幻视严重 / 故事版重点应放窗上）。

## 1. 同行金标在做对的事（逐帧观察）

| 同行做法 | 效果 | 我们 v1 的差距 |
|---|---|---|
| 全片一个真实房间（客厅 + 转角窗），场景 100% 一致 | 可信度高 | v1 成片卧室→办公室→客厅漂移，一眼假 |
| 拉珠珠链**侧边特写**微距实拍（金标第 4 秒） | 「真产品」信任锚点 | 我们不敢给拉珠特写（AI 画不稳），特写缺位 |
| 窗+帘始终是构图主角：人退到前景说话，窗满幅在后景 | 产品记忆点强 | v1 人物大头占画面，窗只占角落（CEO 点名的问题） |
| 生活化剧情钩子（在家看球缺氛围→拉帘变影院感） | 有共鸣、有转化话术 | 我们的 beats 偏「示意」，冲突感弱 |
| 真人对镜说话 + 字幕节奏推进 | UGC 带货感 | AI 人物只能演动作，无口播（暂不追，走产品主角路线） |
| 多窗全景「装好之后」验收镜头 | 电商成交镜头 | v1 有类似 payoff，但因场景漂移不可信 |

## 2. v1 成片缺陷清单（证据帧）

1. **场景漂移**：storyboard 明明锁 same bedroom，成片 2s 卧室 / 8s 办公室 / 13s 客厅——文字锁不住 Seedance。
2. **交叉溶解幻视**：13s 处人物与半透明卷帘叠影（CEO：「幻视还有点儿严重」）。
3. **假尾卡**：原片最后 1s Seedance 幻觉白色名片 + 乱码电话（v1 品牌包装未裁尾，直接拼真尾卡）。
4. **logo 重合风险**：模型偶尔在角落画品牌样文字/标记，与后期左上角烧录 logo 叠一起。
5. **一致性根因**：故事版 3 帧彼此独立生成（只共享文字 Visual Bible），帧间房间/人物本来就不一致，I2V 只能靠溶解硬接——幻视是必然结果。

## 3. 已落地的对策（v2 批次生效）

| 缺陷 | 对策 | 代码 |
|---|---|---|
| 帧间不一致 | **抽卡择优**：每帧 3 候选并行 + vision 评审（窗主角/同房同窗同人/拉珠侧边/零文字）；帧2/3 用**已选前帧作输入图**做图像锚定 | `storyboard-gacha.ts`、两条 pipeline |
| 场景漂移/幻视 | 单场景防幻视锁：一房一窗一机位，禁溶解/叠影/换房；只允许画面内真实运动 | shade/shutter 模板 + I2V prompt |
| 故事版重点 | 窗户主角锁：窗+帘 ≥ 半幅、机位不变、人物配角（CEO 要求写死） | Visual Bible + 评审标准 |
| 假尾卡 | 品牌包装前裁掉原片尾 0.8s 再拼真尾卡 | `tail-trim.ts` |
| logo 重合 | prompt 锁「左上角留白给后期水印」+ 评审对任何模型文字一票否决 + 帧质检记录 | 模板 + `frame-qa.ts` 接入验收脚本 |
| 提交超时误杀（v1 #2/#3 死因） | 提交 fetch 20s 上限 + 3 轮 × 5 plan 重试；轮询超时不再打死任务 | 两条 pipeline |
| shutter 一致性 | 升级为 3 帧故事版（hook/operate/payoff）+ 同套抽卡 | `sunnyshutter-image2-pipeline.ts` |

## 4. 后续提升方向（按优先级）

1. **拉珠侧边特写变体**：抄同行信任锚点——加一个 `chain-closeup` beat（AI 微距风险高，先在故事版抽卡里试，废卡率高就放弃）。
2. **电商三段节奏对齐**：钩子话术更生活化（看球/睡觉/隐私），payoff 加「装好验收」全景镜头。
3. **video-shotcraft（已装 `.claude/skills/video-shotcraft`）**：用其 Remotion 工作流升级品牌包装层——动效尾卡（logo 入场 + 电话数字滚入 + CTA 按钮脉冲）替代静态 3s 底图；其 `references/aesthetic-rules.md`、`music-beat-sync.md` 方法论可并入人工抽检与后期。
4. **Image plan 动态选择**：从 `GET /prices` 拉 `status=available` 的 image plan 替代硬编码列表。
5. **口播/数字人路线**：同行强在真人口播；若客户要口播版，走 HeyGen/OmniHuman 数字人 + 实拍窗景合成，另立验收。
